Spelling shouldn't be a barrier.
So we tested whether it is.

Plenty of us — especially if we're dyslexic — type fast and spell loosely. Talking to an AI feels freeing because it just gets it. This benchmark puts a number on that feeling: we ask each model the same questions twice — once clean, once with realistic messy spelling — and measure how much it still understands.

    How it works

    1

    Ask twice

    Every question is sent to each model in two forms: the original, and a messy version produced by a deterministic "dyslexia-style" corruptor.

    2

    Mess it up realistically

    Not random noise — real patterns: phonetic misspellings, homophone swaps (their/there), transposed and dropped letters, neighbouring-key slips, lost capitals and punctuation.

    3

    Score objectively

    Tasks have checkable answers (multiple choice, short facts, instructions). No AI judge, no opinion — just right or wrong, clean vs messy.

    4

    Report resilience

    Resilience = messy accuracy ÷ clean accuracy. A model that scores the same on both simply doesn't care how you spell.

    Why this matters

    Roughly one in ten people is dyslexic. For a lot of us, the keyboard has always been a gatekeeper: spell it wrong and the search returns nothing, the form rejects you, the room judges you. Large language models quietly removed that gate — they read intent, not orthography.

    That's not a small thing. It means the barrier was never the idea; it was the spelling check in front of it. This project is a tiny, open, reproducible way to show that — with numbers anyone can re-run.
